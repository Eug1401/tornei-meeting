COPPA DEL MONDO + SUPABASE

Questa versione usa Supabase come backend condiviso.
Quando assets/js/supabase-config.js contiene ENABLED:true, Project URL e publishable key:
- le pagine admin richiedono il login Supabase;
- ogni salvataggio admin aggiorna la riga online app_state/main;
- index.html legge la stessa riga online;
- gli utenti pubblici vedono gli aggiornamenti tramite Realtime e polling di sicurezza.

File importanti:
- SUPABASE_SETUP.sql: crea tabella, policy RLS, riga iniziale e configurazione Realtime.
- assets/js/supabase-config.js: contiene Project URL base e publishable key.
- assets/js/supabase-sync.js: gestisce login, salvataggio, Realtime e recupero offline.

Sicurezza:
- La publishable key è pubblica per definizione; non inserire secret/service_role nel frontend.
- La sicurezza delle scritture dipende da Supabase Auth e Row Level Security.
- Le policy incluse permettono SELECT a tutti e scrittura a ogni utente authenticated.
- Se deve esistere un solo admin, disabilita le registrazioni pubbliche oppure restringi le policy al suo auth.uid().

Immagini:
- I loghi squadra e le immagini degli articoli restano nello stato JSON come Data URL.
- Per molti file o immagini pesanti conviene in futuro passare a Supabase Storage.

Articoli:
- Non richiedono nuove tabelle: sono salvati nel campo JSONB data di app_state.
- L'admin li gestisce da admin-articles.html.
- Il pubblico li vede nel tab Articoli di index.html.
